## 1. 오늘의 결론

오늘부터 이 과제는 아래 방향으로 진행한다.

```text
운동 기록 인증 게시판
-> 이전 운동 기록 검색
-> AI가 점진적 과부하 관점으로 비교
-> 다음 운동 목표 추천
```

기술 스택은 아래로 고정한다.

| 영역 | 결정 |
| --- | --- |
| Frontend | React + TypeScript |
| Backend | NestJS + TypeScript |
| AI Server | FastAPI + Python |
| Database | PostgreSQL + pgvector |
| Auth | JWT |
| ORM | Prisma 우선 |
| LLM | GPT-4.1 mini |
| Infra | Docker Compose 우선, AWS는 후반 |

## 4. API 목록 확정

### Auth

| Method | URL | 설명 |
| --- | --- | --- |
| POST | `/auth/signup` | 회원가입 |
| POST | `/auth/login` | 로그인 |
| GET | `/auth/me` | 내 정보 조회 |

`/auth/refresh`는 MVP에서 제외한다. 처음에는 access token 하나로 간다.

### Posts

| Method | URL | 설명 |
| --- | --- | --- |
| GET | `/posts` | 게시글 목록, 검색, 페이징 |
| POST | `/posts` | 게시글 작성 |
| GET | `/posts/:id` | 게시글 상세 |
| PATCH | `/posts/:id` | 게시글 수정 |
| DELETE | `/posts/:id` | 게시글 삭제 |
| GET | `/me/posts` | 내 운동 기록 목록 |

### Comments

| Method | URL | 설명 |
| --- | --- | --- |
| GET | `/posts/:postId/comments` | 댓글 목록 |
| POST | `/posts/:postId/comments` | 댓글 작성 |
| DELETE | `/comments/:id` | 댓글 삭제 |

### Tags

| Method | URL | 설명 |
| --- | --- | --- |
| GET | `/tags` | 태그 목록 |
| GET | `/tags/:name/posts` | 태그별 게시글 |

### AI - React가 호출하는 NestJS API

| Method | URL | 설명 |
| --- | --- | --- |
| POST | `/ai/posts/:postId/analyze` | AI 분석 실행 |

### AI Internal - NestJS가 호출하는 FastAPI API

| Method | URL | 설명 |
| --- | --- | --- |
| GET | `/internal/health` | AI 서버 상태 확인 |

## 6. 브랜치 전략

루트에 Git 저장소를 만들면 아래 방식으로 작업한다.

```text
main
feat/init
feat/backend-posts
feat/auth
feat/frontend-posts
feat/comments-tags-search
feat/ai-server
feat/rag
feat/mcp-agent
docs/readme
```

원칙:

- `main`은 안정된 코드만 둔다.
- 기능 작업은 `feat/...` 브랜치에서 한다.
- 문서 작업은 `docs/...` 브랜치에서 한다.
- 혼자 진행하더라도 이슈와 브랜치를 연결해 작업 흔적을 남긴다.

## 7. Docker Compose 방향

처음에는 DB만 Docker로 띄운다.

```text
1차 Docker Compose
  PostgreSQL + pgvector

나중에 확장
  backend
  ai-server
  frontend
```

이유:

- 처음부터 모든 서버를 Docker로 넣으면 학습 부담이 커진다.
- NestJS, FastAPI, React는 처음에는 로컬에서 직접 실행하는 편이 이해하기 쉽다.
- DB는 설치가 번거로우므로 Docker로 고정한다.

현재 `docker-compose.yml`에는 PostgreSQL + pgvector만 작성했다.
