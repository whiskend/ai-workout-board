backend 폴더에 NestJS 서버를 만들고, GET /health가 응답하며, Prisma가 PostgreSQL에 연결할 준비까지 끝내는 것이다.

6. Health check API 추가

백엔드 서버가 살아 있는지 확인하는 API를 만든다. 배포나 서버 연결 테스트에서 가장 먼저 쓰는 기본 API다.

수정할 파일

backend/src/app.controller.ts

파일 내용을 아래 코드로 교체

import { Controller, Get } from '@nestjs/common';

@Controller()
export class AppController {
  @Get()
  getRoot() {
    return {
      message: 'Workout board backend is running',
    };
  }

  @Get('health')
  getHealth() {
    return {
      status: 'ok',
      service: 'backend',
      timestamp: new Date().toISOString(),
    };
  }
}

다시 서버 실행

cd backend
npm run start:dev

새 터미널에서 확인

curl http://localhost:3000/health

정상 응답 예시

{"status":"ok","service":"backend","timestamp":"2026-06-06T..."}

7. 환경 변수와 ConfigModule 설치

서버 코드 안에 비밀번호나 DB 주소를 직접 적지 않기 위해 환경 변수를 사용한다. NestJS에서는 @nestjs/config를 사용한다.

입력할 명령어

cd backend
npm install @nestjs/config

수정할 파일

backend/src/app.module.ts

파일 내용을 아래 코드로 교체

import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,
    }),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}

정상 기준

서버를 다시 실행했을 때 에러 없이 켜져야 한다.

8. Prisma 설치

Prisma는 NestJS가 PostgreSQL과 대화하기 쉽게 도와주는 도구다. DB 테이블을 코드로 관리하고, 나중에 게시글 CRUD를 만들 때 사용한다.

입력할 명령어

cd backend
npm install @prisma/client
npm install -D prisma
npx prisma init

정상 결과

backend/prisma/schema.prisma
backend/.env

backend/.env 수정

backend/.env 파일의 DATABASE_URL 값을 아래처럼 맞춘다.

DATABASE_URL="postgresql://workout:workout@localhost:5432/workout_board"

Prisma schema 확인

backend/prisma/schema.prisma에 아래와 비슷한 내용이 있어야 한다.

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

9. PrismaService 추가

NestJS 안에서 Prisma를 주입해서 쓰려면 서비스와 모듈을 만든다. 아직 DB 모델은 만들지 않는다. Issue #2에서는 연결 준비까지만 한다.

폴더 만들기

mkdir -p backend/src/prisma

새 파일 만들기

backend/src/prisma/prisma.service.ts

넣을 코드

import { Injectable, OnModuleDestroy, OnModuleInit } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

@Injectable()
export class PrismaService
  extends PrismaClient
  implements OnModuleInit, OnModuleDestroy
{
  async onModuleInit() {
    await this.$connect();
  }

  async onModuleDestroy() {
    await this.$disconnect();
  }
}

새 파일 만들기

backend/src/prisma/prisma.module.ts

넣을 코드

import { Global, Module } from '@nestjs/common';
import { PrismaService } from './prisma.service';

@Global()
@Module({
  providers: [PrismaService],
  exports: [PrismaService],
})
export class PrismaModule {}

app.module.ts 수정

backend/src/app.module.ts를 아래처럼 바꾼다.

import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import { AppController } from './app.controller';
import { AppService } from './app.service';
import { PrismaModule } from './prisma/prisma.module';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,
    }),
    PrismaModule,
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}

10. 서버 실행과 DB 연결 확인

PrismaService는 서버가 켜질 때 DB에 연결을 시도한다. 따라서 DB가 켜져 있고 서버가 정상 실행되면 연결 준비가 된 것이다.

DB 실행 확인

docker compose ps

서버 실행

cd backend
npm run start:dev

health 확인

curl http://localhost:3000/health

정상 기준

서버 실행 중 에러가 없다.
GET /health가 status: ok를 반환한다.

DB 연결 실패 시 자주 나오는 원인

오류 느낌 원인 대처
DB에 연결할 수 없다고 나옴 Docker DB가 꺼져 있음 docker compose up -d 실행
비밀번호 또는 유저 오류 DATABASE_URL이 다름 backend/.env 값 확인
포트 오류 5432 또는 3000 포트 충돌 오류 메시지를 그대로 나에게 보여주기
