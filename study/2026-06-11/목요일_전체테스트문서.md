# 0. 이 문서로 확인하는 것

오늘까지 구현한 기능은 크게 두 묶음이다.

```text
1. 게시글 CRUD
   - 게시글 작성
   - 게시글 목록 조회
   - 게시글 상세 조회
   - 게시글 수정
   - 게시글 삭제

2. 인증 / 권한
   - 회원가입
   - 로그인
   - accessToken 발급
   - token으로 내 정보 조회
   - 로그인한 사용자만 게시글 작성 가능
   - 작성자만 게시글 수정 가능
   - 작성자만 게시글 삭제 가능
```

테스트의 핵심 질문은 이거다.

```text
서버가 켜지는가?

로그인이 되는가?
token이 없으면 막히는가?
token이 있으면 통과되는가?
게시글 작성자가 로그인한 사용자로 저장되는가?
남의 글 수정/삭제가 막히는가?
```

# 7. 6단계: 회원가입 테스트

```bash
curl -X POST http://localhost:3000/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"test1@example.com","password":"12345678","nickname":"테스트1"}'
```

예상:



```json
      {
```
```bash
        "exerciseName": "벤치프레스",
```

```bash
          { "setNumber": 1, "reps": 8 },
          { "setNumber": 2, "reps": 8 },
```

```json
    "id": 1,
```

# 21. 목요일 구현 완료 여부 체크

목요일 학습 문서 기준으로 확인한 결과다.

| 항목 | 상태 | 메모 |
| --- | --- | --- |
| `POST /auth/signup` | 완료 | `AuthController`, `AuthService` 구현됨 |
| `POST /auth/login` | 완료 | bcrypt 비교 후 JWT 발급 |
| `GET /auth/me` | 완료 | `JwtAuthGuard` 적용 |
| `JWT_SECRET` | 완료 | `backend/.env`에 있음 |
| `AuthModule` 등록 | 완료 | `AppModule` imports에 등록됨 |
| 게시글 작성 인증 | 완료 | `POST /posts`에 `JwtAuthGuard` 적용 |
| demo user 제거 | 완료 | `authorId: userId` 사용 |
| 게시글 수정 권한 검사 | 완료 | `post.authorId !== userId` 검사 |
| 게시글 삭제 권한 검사 | 완료 | `post.authorId !== userId` 검사 |
| DTO 검증 적용 | 완료 | `main.ts`에서 `useGlobalPipes`로 수정됨 |
| 빌드 확인 | 완료 | `npm run build` 성공 |

오늘 점검 중 발견해서 고친 것:

```text
1. posts.service.ts
   remove(userid, id)로 되어 있어 내부 userId와 이름이 달랐음
   -> remove(userId, id)로 수정

2. posts.controller.ts
   Request import 위치가 문서와 달랐고 불필요한 import가 있었음
   -> express Request로 정리

3. main.ts
   ValidationPipe가 enableCors 안에 들어가 있었음
   -> app.enableCors()와 app.useGlobalPipes(...)로 분리
```

```text
회원가입 실제 성공
로그인 실제 성공
token 없이 막히는지 확인
token으로 게시글 작성되는지 확인
남의 글 수정/삭제가 403으로 막히는지 확인
```

즉 현재 상태를 정리하면:

```text
코드 구조상 목요일 범위는 완료
빌드도 통과
남은 것은 실제 API 수동 테스트
```

# 23. 최종 체크리스트

아래를 모두 체크하면 오늘 범위는 끝났다고 봐도 된다.

```text
[ ] docker compose up -d 성공
[ ] npm run build 성공
[ ] npm run start:dev 성공
[ ] GET /health 성공
[ ] POST /auth/signup 성공
[ ] POST /auth/login 성공
[ ] GET /auth/me token 없을 때 401
[ ] GET /auth/me token 있을 때 성공
[ ] POST /posts token 없을 때 401
[ ] POST /posts token 있을 때 성공
[ ] GET /posts 성공
[ ] GET /posts/:id 성공
[ ] PATCH /posts/:id 작성자 token으로 성공
[ ] PATCH /posts/:id 다른 사용자 token으로 403
[ ] DELETE /posts/:id 다른 사용자 token으로 403
[ ] DELETE /posts/:id 작성자 token으로 성공
[ ] 삭제 후 GET /posts/:id 404
```
