# 0. 오늘 새로 만나는 문법

오늘 핵심 문법과 개념은 아래다.

```text
bcrypt.hash
bcrypt.compare
JwtService
signAsync
verifyAsync
Authorization header
Bearer token
@UseGuards
CanActivate
ExecutionContext
@Req
Request 타입 확장
ConflictException
UnauthorizedException
ForbiddenException
```

오늘의 핵심 흐름:

```text
비밀번호를 hash한다.
        ↓
로그인 때 비밀번호를 비교한다.
        ↓
JWT를 만든다.
        ↓
요청 header에서 JWT를 꺼낸다.
        ↓
JWT를 검증한다.
        ↓
request.user에 사용자 정보를 넣는다.
        ↓
Controller에서 request.user를 사용한다.
```

---

# 1. bcrypt.hash

비밀번호는 DB에 그대로 저장하면 안 된다.

나쁜 예:

```ts
password: 'password123'
```

좋은 예:

```ts
const passwordHash = await bcrypt.hash(signupDto.password, 10);
```

뜻:

```text
사용자가 입력한 비밀번호를 복원하기 어려운 문자열로 바꾼다.
```

`10`은 salt rounds다.

처음에는 이렇게 이해하면 된다.

```text
숫자가 클수록 더 오래 걸리고 강해진다.
과제에서는 10이면 충분하다.
```

---

# 2. bcrypt.compare

로그인할 때는 사용자가 입력한 비밀번호와 DB에 저장된 hash를 비교한다.

```ts
const isPasswordValid = await bcrypt.compare(
  loginDto.password,
  user.passwordHash,
);
```

뜻:

```text
사용자가 입력한 비밀번호가
저장된 passwordHash와 같은 비밀번호에서 나온 것인지 확인한다.
```

결과:

```text
true  = 비밀번호 맞음
false = 비밀번호 틀림
```

---

# 3. JwtService

`JwtService`는 JWT를 만들고 검증하는 객체다.

```ts
constructor(private readonly jwtService: JwtService) {}
```

여기서 `jwtService`는 NestJS가 주입해준다.

사용하려면 `AuthModule`에서 `JwtModule`을 등록해야 한다.

```ts
JwtModule.registerAsync(...)
```

---

# 4. signAsync

JWT를 만들 때 쓴다.

```ts
const accessToken = await this.jwtService.signAsync({
  sub: user.id,
  email: user.email,
  nickname: user.nickname,
});
```

뜻:

```text
이 사용자 정보를 token 안에 넣고,
JWT_SECRET으로 서명해서,
accessToken 문자열을 만든다.
```

여기서 `sub`는 subject의 줄임말이다.

```text
sub = 이 token의 주인
```

보통 사용자 id를 넣는다.

---

# 5. verifyAsync

JWT를 검증할 때 쓴다.

```ts
const payload = await this.jwtService.verifyAsync(token);
```

뜻:

```text
이 token이 우리 서버가 만든 token인지 확인한다.
만료되지 않았는지 확인한다.
정상이면 token 안의 payload를 꺼낸다.
```

검증에 실패하면 catch로 간다.

```ts
try {
  const payload = await this.jwtService.verifyAsync(token);
} catch {
  throw new UnauthorizedException('유효하지 않은 토큰입니다.');
}
```

---

# 6. Authorization header

로그인한 사용자는 요청 header에 token을 담아 보낸다.

형식:

```http
Authorization: Bearer <token>
```

예:

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

여기서:

```text
Authorization = 인증 정보를 담는 HTTP header 이름
Bearer        = token 인증 방식 표시
token         = 실제 JWT 문자열
```

---

# 7. header에서 token 꺼내기

코드:

```ts
private extractTokenFromHeader(request: Request): string | undefined {
  const [type, token] = request.headers.authorization?.split(' ') ?? [];
  return type === 'Bearer' ? token : undefined;
}
```

예:

```ts
'Bearer abc.def.ghi'.split(' ')
```

결과:

```ts
['Bearer', 'abc.def.ghi']
```

그래서:

```ts
type = 'Bearer'
token = 'abc.def.ghi'
```

---

# 8. @UseGuards

`@UseGuards`는 Controller 함수 실행 전에 Guard를 먼저 실행하라는 뜻이다.

```ts
@UseGuards(JwtAuthGuard)
@Get('me')
getMe() {}
```

흐름:

```text
GET /auth/me 요청
        ↓
JwtAuthGuard 먼저 실행
        ↓
token 검사 성공
        ↓
getMe 실행
```

token 검사 실패:

```text
401 Unauthorized
```

---

# 9. CanActivate

Guard는 `CanActivate`를 구현한다.

```ts
export class JwtAuthGuard implements CanActivate {
  async canActivate(context: ExecutionContext): Promise<boolean> {
    return true;
  }
}
```

뜻:

```text
이 클래스는 요청을 통과시킬지 막을지 판단할 수 있다.
```

`canActivate` 결과:

```text
true  = 통과
false = 차단
throw = 차단 + 에러 응답
```

---

# 10. ExecutionContext

Guard 안에서는 현재 요청을 직접 매개변수로 받지 않는다.

대신 `ExecutionContext`에서 꺼낸다.

```ts
const request = context.switchToHttp().getRequest<Request>();
```

뜻:

```text
현재 실행 중인 요청을 HTTP 요청으로 보고,
그 request 객체를 꺼낸다.
```

이 request에서 header를 읽는다.

```ts
request.headers.authorization
```

---

# 11. request.user

JWT 검증에 성공하면 request에 user 정보를 붙인다.

```ts
request['user'] = {
  userId: payload.sub,
  email: payload.email,
  nickname: payload.nickname,
};
```

이렇게 하면 Controller에서 사용할 수 있다.

```ts
@Req() request: AuthenticatedRequest
```

```ts
request.user.userId
```

뜻:

```text
현재 로그인한 사용자의 id
```

---

# 12. Request 타입 확장

기본 `Request` 타입에는 `user`가 없다.

그래서 TypeScript에게 알려줘야 한다.

```ts
type AuthenticatedRequest = Request & {
  user: {
    userId: number;
    email: string;
    nickname: string;
  };
};
```

여기서 `&`는 타입을 합친다는 뜻이다.

```text
Request이면서 user도 가진 타입
```

---

# 13. @Req

`@Req()`는 request 객체를 Controller 함수에서 받겠다는 뜻이다.

```ts
getMe(@Req() request: AuthenticatedRequest) {
  return request.user;
}
```

뜻:

```text
현재 HTTP 요청 객체를 request 변수로 받는다.
그 안의 user를 반환한다.
```

# 14. Exception 종류

## 14.1 ConflictException

```ts
throw new ConflictException('이미 가입된 이메일입니다.');
```

뜻:

```text
409 Conflict
```

이미 가입된 이메일로 다시 가입하려 할 때 사용한다.

## 14.2 UnauthorizedException

```ts
throw new UnauthorizedException('이메일 또는 비밀번호가 올바르지 않습니다.');
```

뜻:

```text
401 Unauthorized
```

로그인 실패, token 없음, token 오류에 사용한다.

## 14.3 ForbiddenException

```ts
throw new ForbiddenException('본인이 작성한 게시글만 수정/삭제할 수 있습니다.');
```

뜻:

```text
403 Forbidden
```

로그인은 했지만 권한이 없을 때 사용한다.

정리:

```text
401 = 너 누구야? 로그인부터 해.
403 = 누군지는 아는데 이 작업은 못 해.
409 = 이미 있어서 충돌이야.
404 = 찾는 데이터가 없어.
```

# 15. 작성자 권한 검사

게시글에는 작성자 id가 있다.

```ts
post.authorId
```

JWT token에는 현재 사용자 id가 있다.

```ts
request.user.userId
```

둘을 비교한다.

```ts
if (post.authorId !== userId) {
  throw new ForbiddenException('본인이 작성한 게시글만 수정/삭제할 수 있습니다.');
}
```

뜻:

```text
게시글 작성자와 현재 로그인 사용자가 다르면 막는다.
```
