## 1. 현재 상태 판단

현재 작업은 원래 로드맵 기준으로 보면 **화요일 작업까지 온 상태**다.

완료된 것:

```text
POST /posts
GET /posts
GET /posts/:id
PATCH /posts/:id
DELETE /posts/:id
```

즉 게시글 CRUD API는 코드상 들어와 있다.

하지만 수요일 목표였던 아래 작업은 아직 못 했다.

```text
회원가입
로그인
JWT 발급
JWT Guard
내 정보 조회
작성자 권한 처리
React API 연결
```

그래서 오늘의 결론은 이것이다.

```text
목요일 원래 계획인 댓글/태그/검색/페이징으로 넘어가지 않는다.
수요일에 못 한 인증/JWT를 오늘 먼저 한다.
```

## 2. 오늘 확정할 방향

오늘 작업의 핵심은 **사용자 구분**이다.

현재는 게시글을 작성할 때 demo user를 사용한다.

현재 흐름:

```text
POST /posts
        ↓
getDemoUser()
        ↓
demo@board.local 사용자로 게시글 작성
```

오늘 끝나야 하는 흐름:

```text
회원가입
        ↓
로그인
        ↓
JWT accessToken 발급
        ↓
POST /posts 요청에 Authorization header 포함
        ↓
NestJS가 토큰 확인
        ↓
토큰 속 userId로 게시글 작성
```

## 6. 오늘 성공 기준

오늘 성공 기준은 아래다.

```bash
curl -X POST /auth/signup
```

으로 회원가입이 된다.

```bash
curl -X POST /auth/login
```

으로 accessToken을 받는다.

```bash
curl GET /auth/me \
  -H "Authorization: Bearer <token>"
```

으로 내 정보를 조회한다.

```bash
curl -X POST /posts \
  -H "Authorization: Bearer <token>"
```

으로 로그인한 사용자 id로 게시글을 작성한다.

```bash
PATCH /posts/:id
DELETE /posts/:id
```

에서 작성자만 수정/삭제 가능하다.
