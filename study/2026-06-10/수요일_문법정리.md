# 0. 오늘 새로 만나는 문법

월요일에는 아래 문법을 많이 봤다.

```text
@Controller
@Get
@Post
@Body
@Param
@Injectable
Prisma create/findMany/findUnique
DTO
```

수요일에는 여기에 아래가 추가된다.

```text
@Patch
@Delete
@UseGuards
CanActivate
ExecutionContext
Authorization header
Bearer token
JWT sign / verify
bcrypt hash / compare
PartialType
localStorage
fetch
useState
useEffect
form onSubmit
```

오늘 문법의 핵심은 이것이다.

```text
요청을 받는다.
        ↓
토큰을 검사한다.
        ↓
누가 요청했는지 알아낸다.
        ↓
그 사용자가 할 수 있는 작업인지 확인한다.
        ↓
DB 작업을 한다.
        ↓
React 화면에 결과를 보여준다.
```

# 1. PATCH 문법

## 1.1 PATCH는 무엇인가?

`PATCH`는 HTTP method 중 하나다.

보통 **일부 수정**에 사용한다.

예:

```text
PATCH /posts/1
```

뜻:

```text
id가 1인 게시글의 일부 값을 수정한다.
```

NestJS에서는 이렇게 쓴다.

```ts
@Patch(':id')
update(@Param('id', ParseIntPipe) id: number) {
  return this.postsService.update(id);
}
```

여기서 중요한 것:

| 코드 | 의미 |
| --- | --- |
| `@Patch(':id')` | PATCH `/posts/:id` 요청 담당 |
| `@Param('id')` | URL의 `id` 값을 꺼냄 |
| `ParseIntPipe` | 문자열 id를 숫자로 변환 |
| `update()` | 우리가 만든 함수 이름 |

함수 이름은 꼭 `update`일 필요는 없다.

중요한 건 `@Patch(':id')`가 붙었다는 점이다.

# 2. DELETE 문법

## 2.1 DELETE는 무엇인가?

`DELETE`는 HTTP method 중 하나다.

보통 데이터를 삭제할 때 사용한다.

예:

```text
DELETE /posts/1
```

뜻:

```text
id가 1인 게시글을 삭제한다.
```

NestJS에서는 이렇게 쓴다.

```ts
@Delete(':id')
remove(@Param('id', ParseIntPipe) id: number) {
  return this.postsService.remove(id);
}
```

정리:

```text
@Get       조회
@Post      생성
@Patch     수정
@Delete    삭제
```

게시판 CRUD와 연결하면:

```text
Create  = POST
Read    = GET
Update  = PATCH
Delete  = DELETE
```

# 3. PartialType 문법

## 3.1 왜 필요한가?

게시글 생성 DTO는 보통 필수 값이 많다.

```ts
export class CreatePostDto {
  title: string;
  date: string;
  bodyPart: string;
  exercises: CreateExerciseDto[];
}
```

생성할 때는 제목, 날짜, 운동 부위, 운동 목록이 있어야 한다.

하지만 수정할 때는 제목만 바꿀 수도 있다.

```json
{
  "title": "수정된 제목"
}
```

그래서 update DTO에서는 모든 필드가 선택 사항이어야 한다.

## 3.2 PartialType 사용

```ts
import { PartialType } from '@nestjs/mapped-types';
import { CreatePostDto } from './create-post.dto';

export class UpdatePostDto extends PartialType(CreatePostDto) {}
```

뜻:

```text
CreatePostDto의 모든 필드를 optional로 바꾼 새 DTO를 만든다.
```

즉 아래를 자동으로 만들어주는 느낌이다.

```ts
export class UpdatePostDto {
  title?: string;
  date?: string;
  bodyPart?: string;
  exercises?: CreateExerciseDto[];
}
```

여기서 `?`는 없어도 되는 값이라는 뜻이다.

# 4. optional `?` 문법

TypeScript에서 `?`는 선택 사항이라는 뜻이다.

```ts
memo?: string;
```

뜻:

```text
memo는 string일 수도 있고,
아예 안 들어올 수도 있다.
```

아래 둘 다 가능하다.

```json
{
  "memo": "마지막 세트 힘들었다"
}
```

```json
{}
```

수정 API에서 특히 중요하다.

```ts
title?: string;
memo?: string;
```

뜻:

```text
수정 요청에서 title만 들어와도 되고,
memo만 들어와도 되고,
둘 다 들어와도 된다.
```

# 5. undefined와 null

오늘 Prisma update를 하다 보면 `undefined`를 보게 된다.

```ts
date: updatePostDto.date
  ? new Date(updatePostDto.date)
  : undefined,
```

느낌:

```text
값이 있으면 Date로 바꿔서 수정한다.
값이 없으면 undefined를 넣는다.
```

Prisma update에서 보통:

```text
undefined = 이 필드는 건드리지 않음
null      = 이 필드를 비움
```

예:

```ts
memo: undefined
```

뜻:

```text
memo는 수정하지 않는다.
```

```ts
memo: null
```

뜻:

```text
memo를 비운다.
```

처음에는 이렇게 기억하면 된다.

```text
undefined는 "안 건드림"
null은 "비움"
```

# 6. @UseGuards 문법

## 6.1 Guard는 무엇인가?

Guard는 요청이 Controller 함수에 도착하기 전에 막거나 통과시키는 문지기다.

```ts
@UseGuards(JwtAuthGuard)
@Get('me')
getMe() {}
```

뜻:

```text
GET /auth/me 요청이 오면
getMe()를 바로 실행하지 말고
먼저 JwtAuthGuard를 통과해야 한다.
```

흐름:

```text
요청 들어옴
        ↓
JwtAuthGuard 실행
        ↓
토큰 있음? 유효함?
        ↓
통과하면 Controller 실행
        ↓
실패하면 401 Unauthorized 응답
```

## 6.2 어디에 붙이나?

메서드 하나에 붙일 수 있다.

```ts
@UseGuards(JwtAuthGuard)
@Post()
create() {}
```

클래스 전체에 붙일 수도 있다.

```ts
@UseGuards(JwtAuthGuard)
@Controller('posts')
export class PostsController {}
```

클래스에 붙이면 그 컨트롤러 안의 모든 API가 보호된다.

처음에는 메서드마다 붙이는 게 더 이해하기 쉽다.

# 7. CanActivate 문법

Guard 클래스는 보통 `CanActivate`를 구현한다.

```ts
export class JwtAuthGuard implements CanActivate {
  async canActivate(context: ExecutionContext): Promise<boolean> {
    return true;
  }
}
```

뜻:

```text
이 클래스는 요청을 통과시킬지 말지 판단할 수 있는 Guard다.
```

`canActivate()`가:

```text
true 반환  = 요청 통과
false 반환 = 요청 차단
예외 throw = 요청 차단 + 에러 응답
```

JWT가 없으면:

```ts
throw new UnauthorizedException('인증 토큰이 없습니다.');
```

그러면 NestJS가 HTTP 401 응답을 보낸다.

# 8. ExecutionContext 문법

Guard 안에서는 Controller 함수의 매개변수를 바로 받을 수 없다.

대신 `ExecutionContext`를 통해 현재 요청 객체를 꺼낸다.

```ts
const request = context.switchToHttp().getRequest<Request>();
```

뜻:

```text
현재 실행 중인 요청은 HTTP 요청이다.
그 HTTP 요청 객체를 꺼낸다.
```

이 request 안에 header가 있다.

```ts
request.headers.authorization
```

여기서 JWT 토큰을 꺼낸다.

# 9. Authorization header 문법

로그인한 사용자는 API 요청을 보낼 때 토큰을 같이 보내야 한다.

형식:

```http
Authorization: Bearer <token>
```

예:

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

여기서:

```text
Authorization = 인증 정보를 담는 HTTP header 이름
Bearer        = 토큰 방식이라는 표시
token         = 실제 JWT 문자열
```

Guard에서는 이렇게 꺼낸다.

```ts
const [type, token] = request.headers.authorization?.split(' ') ?? [];
return type === 'Bearer' ? token : undefined;
```

예:

```text
"Bearer abc.def.ghi".split(' ')
```

결과:

```ts
["Bearer", "abc.def.ghi"]
```

그래서:

```ts
type = "Bearer"
token = "abc.def.ghi"
```

# 10. JWT 문법

## 10.1 JWT는 무엇인가?

JWT는 로그인 후 서버가 발급해주는 문자열 토큰이다.

토큰 안에는 보통 이런 정보가 들어간다.

```ts
{
  sub: user.id,
  email: user.email,
  nickname: user.nickname
}
```

여기서 `sub`는 subject의 줄임말이다.

보통 JWT에서 사용자의 id를 넣는 표준적인 이름으로 많이 쓴다.

```text
sub = 이 토큰의 주인
```

## 10.2 토큰 만들기

```ts
const accessToken = await this.jwtService.signAsync({
  sub: user.id,
  email: user.email,
  nickname: user.nickname,
});
```

뜻:

```text
사용자 정보를 payload로 넣고
JWT_SECRET으로 서명해서
accessToken 문자열을 만든다.
```

## 10.3 토큰 검증

```ts
const payload = await this.jwtService.verifyAsync(token);
```

뜻:

```text
이 토큰이 진짜 우리 서버가 만든 토큰인지 확인한다.
만료되지 않았는지 확인한다.
정상이면 payload를 꺼낸다.
```

검증 실패 시:

```ts
throw new UnauthorizedException('유효하지 않은 토큰입니다.');
```

# 11. bcrypt 문법

## 11.1 왜 bcrypt를 쓰나?

비밀번호는 DB에 그대로 저장하면 안 된다.

나쁜 예:

```text
password = "password123"
```

DB가 털리면 사용자의 비밀번호가 그대로 노출된다.

그래서 비밀번호를 hash로 바꿔 저장한다.

```ts
const passwordHash = await bcrypt.hash(password, 10);
```

## 11.2 hash

```ts
await bcrypt.hash(signupDto.password, 10);
```

뜻:

```text
사용자가 입력한 비밀번호를 복원하기 어려운 문자열로 바꾼다.
```

`10`은 salt rounds다.

처음에는 이렇게 이해하면 된다.

```text
숫자가 클수록 더 오래 걸리고 조금 더 강하다.
개발/일반 과제에서는 10 정도면 충분하다.
```

## 11.3 compare

로그인할 때는 사용자가 입력한 비밀번호와 DB의 hash를 비교해야 한다.

```ts
const isPasswordValid = await bcrypt.compare(
  loginDto.password,
  user.passwordHash,
);
```

뜻:

```text
입력한 비밀번호가
저장된 passwordHash와 같은 비밀번호에서 나온 것인지 확인한다.
```

같으면 `true`, 다르면 `false`다.

# 12. Exception 문법

NestJS에서는 에러를 throw하면 HTTP 응답으로 바꿔준다.

## 12.1 ConflictException

```ts
throw new ConflictException('이미 가입된 이메일입니다.');
```

뜻:

```text
409 Conflict 응답
```

이미 존재하는 이메일로 회원가입하려 할 때 사용한다.

## 12.2 UnauthorizedException

```ts
throw new UnauthorizedException('이메일 또는 비밀번호가 올바르지 않습니다.');
```

뜻:

```text
401 Unauthorized 응답
```

로그인이 안 됐거나 토큰이 잘못됐을 때 사용한다.

## 12.3 ForbiddenException

```ts
throw new ForbiddenException('본인이 작성한 게시글만 수정할 수 있습니다.');
```

뜻:

```text
403 Forbidden 응답
```

로그인은 했지만 권한이 없을 때 사용한다.

차이:

```text
401 Unauthorized = 너 누구야? 로그인부터 해.
403 Forbidden    = 네가 누군지는 아는데, 이 작업은 못 해.
404 Not Found    = 요청한 데이터가 없어.
409 Conflict     = 이미 있어서 충돌이 나.
```

# 13. Request 타입 문법

Controller에서 request를 직접 받을 수 있다.

```ts
@Req() request
```

하지만 TypeScript에서는 request 안에 어떤 값이 있는지 알려주는 게 좋다.

```ts
type AuthenticatedRequest = Request & {
  user: {
    userId: number;
    email: string;
    nickname: string;
  };
};
```

뜻:

```text
기본 Request 타입에
user라는 값을 추가로 가지고 있는 타입이다.
```

`&`는 타입을 합친다는 뜻이다.

```ts
Request & { user: ... }
```

뜻:

```text
Request이면서 user도 가진 객체
```

# 14. React useState 문법

React에서 입력값을 저장할 때 `useState`를 쓴다.

```tsx
const [email, setEmail] = useState('');
```

뜻:

```text
email이라는 상태값을 만든다.
처음 값은 빈 문자열이다.
email 값을 바꿀 때는 setEmail을 사용한다.
```

입력창과 연결:

```tsx
<input
  value={email}
  onChange={(event) => setEmail(event.target.value)}
/>
```

흐름:

```text
사용자가 input에 글자를 입력한다.
        ↓
onChange 실행
        ↓
setEmail로 상태 변경
        ↓
email 값이 바뀐다.
        ↓
화면이 다시 렌더링된다.
```

# 15. React FormEvent 문법

폼 제출 이벤트 타입:

```tsx
import { FormEvent } from 'react';
```

사용:

```tsx
async function handleSubmit(event: FormEvent<HTMLFormElement>) {
  event.preventDefault();
}
```

`event.preventDefault()`는 중요하다.

기본 HTML form은 submit하면 페이지를 새로고침한다.

React 앱에서는 새로고침하면 상태가 날아가기 때문에 막아야 한다.

```text
event.preventDefault()
= form의 기본 새로고침 동작을 막는다.
```

# 16. fetch 문법

브라우저에서 API 요청을 보낼 때 `fetch`를 쓴다.

```ts
const response = await fetch('http://localhost:3000/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email,
    password,
  }),
});
```

뜻:

```text
POST /auth/login 요청을 보낸다.
body는 JSON 문자열로 보낸다.
```

중요:

```ts
JSON.stringify(...)
```

JavaScript 객체를 JSON 문자열로 바꾼다.

```ts
{ email: 'a@test.com' }
```

이 객체를 HTTP body로 보내려면 문자열로 바꿔야 한다.

```json
{"email":"a@test.com"}
```

응답 읽기:

```ts
const result = await response.json();
```

뜻:

```text
서버가 보낸 JSON 응답을 JavaScript 객체로 바꾼다.
```

# 17. localStorage 문법

로그인 후 받은 JWT를 브라우저에 저장해야 한다.

가장 단순한 방법은 `localStorage`다.

저장:

```ts
localStorage.setItem('accessToken', token);
```

읽기:

```ts
localStorage.getItem('accessToken');
```

삭제:

```ts
localStorage.removeItem('accessToken');
```

주의:

```text
localStorage는 편하지만 완벽하게 안전한 저장소는 아니다.
```

하지만 이번 과제 MVP에서는 충분하다.

나중에 보안을 더 챙기려면 httpOnly cookie 같은 방식을 고려한다.

# 18. Authorization header를 fetch에 넣기

로그인한 API 요청은 header에 토큰을 넣어야 한다.

```ts
const token = localStorage.getItem('accessToken');

await fetch('http://localhost:3000/posts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Authorization: `Bearer ${token}`,
  },
  body: JSON.stringify(data),
});
```

여기서 백틱 문법:

```ts
`Bearer ${token}`
```

뜻:

```text
Bearer 뒤에 token 변수 값을 붙인 문자열을 만든다.
```

예:

```text
Bearer abc.def.ghi
```

# 19. useEffect 문법

페이지가 열릴 때 API를 한 번 호출하고 싶으면 `useEffect`를 쓴다.

```tsx
useEffect(() => {
  async function loadPosts() {
    const data = await apiFetch('/posts');
    setPosts(data);
  }

  loadPosts();
}, []);
```

여기서 마지막 `[]`가 중요하다.

```text
[] = 이 컴포넌트가 처음 화면에 나타날 때 한 번만 실행
```

흐름:

```text
PostListPage가 화면에 나타난다.
        ↓
useEffect 실행
        ↓
GET /posts 요청
        ↓
응답을 posts state에 저장
        ↓
화면에 목록 출력
```

# 20. map 문법

배열을 화면에 출력할 때 `map`을 쓴다.

```tsx
{posts.map((post) => (
  <article key={post.id}>
    <h2>{post.title}</h2>
    <p>{post.bodyPart}</p>
  </article>
))}
```

뜻:

```text
posts 배열의 각 post를
article JSX로 바꿔서 화면에 보여준다.
```

`key`는 React가 목록을 구분하기 위해 필요하다.

```tsx
key={post.id}
```

뜻:

```text
이 목록 아이템의 고유 식별자는 post.id다.
```

# 21. useNavigate 문법

코드에서 페이지 이동을 하고 싶으면 `useNavigate`를 쓴다.

```tsx
const navigate = useNavigate();
```

사용:

```tsx
navigate('/posts');
```

뜻:

```text
/posts 페이지로 이동한다.
```

로그인 후:

```tsx
saveAccessToken(result.accessToken);
navigate('/posts');
```

게시글 작성 후:

```tsx
navigate(`/posts/${createdPost.id}`);
```

# 22. React Router path param

상세 페이지 주소는 보통 이렇게 만든다.

```tsx
<Route path="/posts/:id" element={<PostDetailPage />} />
```

여기서 `:id`는 변하는 값이다.

예:

```text
/posts/1
/posts/2
/posts/100
```

상세 페이지에서 id를 꺼내려면:

```tsx
const { id } = useParams();
```

API 요청:

```tsx
const post = await apiFetch(`/posts/${id}`);
```

# 23. 오늘 문법 한 장 요약

```text
@Patch(':id')
= PATCH /posts/:id 요청 담당

@Delete(':id')
= DELETE /posts/:id 요청 담당

PartialType(CreateDto)
= 생성 DTO를 수정 DTO로 재사용

@UseGuards(JwtAuthGuard)
= Controller 실행 전에 JWT 검사

CanActivate
= Guard가 요청 통과 여부를 판단하는 메서드 규칙

Authorization: Bearer token
= 로그인한 사용자가 토큰을 보내는 HTTP header

jwtService.signAsync()
= JWT 토큰 만들기

jwtService.verifyAsync()
= JWT 토큰 검증하기

bcrypt.hash()
= 비밀번호를 hash로 바꾸기

bcrypt.compare()
= 입력 비밀번호와 hash 비교

localStorage
= 브라우저에 토큰 저장

fetch
= 브라우저에서 API 요청 보내기

useState
= React 상태 저장

useEffect
= 화면이 열릴 때 API 요청

useNavigate
= 코드로 페이지 이동
```
