# 0. 오늘 새로 만나는 문법

오늘 핵심 문법과 개념은 아래다.

```text
FastAPI
APIRouter
Pydantic BaseModel
response_model
Python list type
Python None
uvicorn
NestJS service에서 fetch
ConfigService
TypeScript type export
Prisma not / in / take
map 변환 함수
disabled button
로딩 state
```

오늘의 핵심 흐름:

```text
FastAPI에 분석 요청 형식을 만든다.
        ↓
FastAPI에 분석 API를 만든다.
        ↓
NestJS가 FastAPI를 fetch로 호출한다.
        ↓
NestJS가 현재 게시글과 이전 기록을 묶어서 보낸다.
        ↓
React가 NestJS 분석 API를 호출한다.
        ↓
분석 결과를 state에 저장하고 화면에 보여준다.
```

# 1. FastAPI

FastAPI는 Python으로 API 서버를 만들 때 쓰는 프레임워크다.

예:

```py
from fastapi import FastAPI

app = FastAPI()
```

뜻:

```text
FastAPI 서버 앱을 하나 만든다.
```

NestJS에서 `NestFactory.create(AppModule)`로 서버를 만들었다면,  
FastAPI에서는 `FastAPI()`로 서버 앱을 만든다고 보면 된다.

---

# 2. APIRouter

`APIRouter`는 API 주소들을 파일별로 나누기 위한 도구다.

예:

```py
from fastapi import APIRouter

router = APIRouter()
```

이렇게 만든 router에 API를 붙인다.

```py
@router.post("/demo")
def analyze_demo():
    return {"message": "ok"}
```

이건 NestJS의 Controller와 비슷하게 생각하면 된다.

```text
NestJS Controller
-> 요청 주소를 모아둔 클래스

FastAPI APIRouter
-> 요청 주소를 모아둔 router 객체
```

---

# 3. Pydantic BaseModel

FastAPI에서는 요청 body의 모양을 검사할 때 `BaseModel`을 자주 쓴다.

예:

```py
from pydantic import BaseModel


class ChatRequest(BaseModel):
    message: str
```

뜻:

```text
이 API는 message라는 문자열을 받아야 한다.
```

NestJS의 DTO와 비슷하다.

```text
NestJS DTO
-> class CreatePostDto

FastAPI Pydantic schema
-> class AnalysisRequest(BaseModel)
```

둘 다 역할은 비슷하다.

```text
요청 데이터가 약속한 모양인지 확인한다.
```

---

# 4. response_model

FastAPI에서 응답 모양을 정할 때 `response_model`을 쓴다.

예:

```py
@router.post("/demo", response_model=AnalysisResponse)
def analyze_demo(request: AnalysisRequest):
    return make_demo_analysis(request)
```

뜻:

```text
이 API는 AnalysisResponse 모양으로 응답한다.
```

좋은 점:

```text
응답 데이터 모양을 문서화할 수 있다.
응답에 필요 없는 값이 섞이는 것을 줄일 수 있다.
FastAPI docs에서 응답 구조를 볼 수 있다.
```

# 5. Python 타입 표기

Python에서도 타입을 적을 수 있다.

```py
message: str
count: int
weight: float
```

뜻:

```text
message는 문자열
count는 정수
weight는 실수
```

배열은 이렇게 쓴다.

```py
sets: list[SetRecord]
```

뜻:

```text
sets는 SetRecord 여러 개가 들어 있는 배열이다.
```

TypeScript로 치면:

```ts
sets: SetRecord[]
```

와 비슷하다.

# 6. None과 Optional

Python에서 값이 없음을 나타낼 때 `None`을 쓴다.

```py
from typing import Optional

memo: Optional[str] = None
```

뜻:

```text
memo는 문자열일 수도 있고,
없으면 None일 수도 있다.
기본값은 None이다.
```

현재 프로젝트의 Python은 3.9 계열이다.

그래서 아래 문법은 쓰지 않는다.

```py
memo: str | None = None
```

이 문법은 Python 3.10 이상에서 쓰는 방식이다.

TypeScript로 치면:

```ts
memo?: string | null;
```

와 비슷하게 보면 된다.

# 7. uvicorn

`uvicorn`은 FastAPI 앱을 실제 서버로 실행해주는 도구다.

명령어:

```bash
uvicorn app.main:app --reload --port 8000
```

뜻:

```text
app/main.py 파일 안에 있는 app 객체를 실행한다.
파일이 바뀌면 자동으로 다시 실행한다.
8000번 포트로 서버를 연다.
```

분해:

```text
app.main
-> app/main.py

:app
-> main.py 안의 app 변수

--reload
-> 코드 변경 시 자동 재시작

--port 8000
-> localhost:8000에서 실행
```

# 8. NestJS service에서 fetch

금요일에는 React에서 fetch를 썼다.

토요일에는 NestJS에서도 fetch를 쓴다.

```ts
const response = await fetch(`${aiServerUrl}/analysis/demo`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify(payload),
});
```

뜻:

```text
NestJS 백엔드가 FastAPI AI 서버로 HTTP 요청을 보낸다.
```

흐름:

```text
React
-> NestJS
-> fetch
-> FastAPI
```

즉 fetch는 프론트에서만 쓰는 것이 아니다.

서버 코드에서도 다른 서버에 HTTP 요청을 보낼 때 쓸 수 있다.

# 9. ConfigService

`ConfigService`는 `.env` 값을 코드에서 읽기 위한 NestJS 도구다.

예:

```ts
constructor(private readonly configService: ConfigService) {}
```

```ts
const aiServerUrl =
  this.configService.get<string>('AI_SERVER_URL') ?? 'http://localhost:8000';
```

뜻:

```text
.env에 있는 AI_SERVER_URL 값을 읽는다.
없으면 http://localhost:8000을 쓴다.
```

왜 쓰나?

```text
개발 환경에서는 localhost:8000
배포 환경에서는 실제 AI 서버 주소
```

처럼 주소가 바뀔 수 있기 때문이다.

# 10. TypeScript type export

타입을 다른 파일에서 쓰게 하려면 `export`를 붙인다.

```ts
export type AnalysisResult = {
  summary: string;
  recommendation: string;
  nextGoal: string;
  referencedPostCount: number;
};
```

다른 파일에서 가져올 때:

```ts
import type { AnalysisResult } from '../types/analysis';
```

뜻:

```text
AnalysisResult라는 타입만 가져온다.
실제 JavaScript 값은 가져오지 않는다.
```

`import type`은 타입 전용 import다.

# 11. Prisma `not`

Prisma에서 특정 값이 아닌 데이터를 찾을 때 `not`을 쓴다.

```ts
id: {
  not: id,
}
```

뜻:

```text
현재 게시글 id와 다른 게시글만 찾는다.
```

왜 필요하나?

```text
현재 게시글과 비교할 이전 기록을 찾아야 한다.
현재 게시글 자기 자신은 이전 기록이 아니다.
```

# 12. Prisma `in`

여러 값 중 하나에 포함되는지 확인할 때 `in`을 쓴다.

```ts
normalizedName: {
  in: normalizedNames,
}
```

뜻:

```text
normalizedName이 normalizedNames 배열 안에 포함되는 게시글을 찾는다.
```

예:

```ts
normalizedNames = ['벤치프레스', '인클라인덤벨프레스'];
```

그러면:

```text
운동명이 벤치프레스이거나
운동명이 인클라인덤벨프레스인 기록을 찾는다.
```

# 13. Prisma `take`

`take`는 몇 개만 가져올지 정하는 옵션이다.

```ts
take: 3
```

뜻:

```text
최대 3개만 가져온다.
```

왜 필요한가?

```text
이전 기록이 너무 많으면 AI 서버에 보낼 데이터가 커진다.
일단 최근 기록 3개만 비교 재료로 사용한다.
```

# 14. 변환 함수

Prisma에서 가져온 객체를 그대로 FastAPI로 보내지 않고, 필요한 모양으로 바꾼다.

```ts
function toAiPostRecord(post: any): AiPostRecord {
  return {
    id: post.id,
    title: post.title,
    date: post.date.toISOString().slice(0, 10),
    bodyPart: post.bodyPart,
    memo: post.memo,
    exercises: post.exercises.map((exercise) => ({
      exerciseName: exercise.exerciseName,
      weightKg: exercise.weightKg,
      targetReps: exercise.targetReps,
      sets: exercise.sets.map((set) => ({
        setNumber: set.setNumber,
        reps: set.reps,
        perceivedDifficulty: set.perceivedDifficulty,
      })),
    })),
  };
}
```

뜻:

```text
DB에서 꺼낸 복잡한 post 객체를
AI 서버에 보내기 좋은 단순 JSON으로 바꾼다.
```

이런 함수를 만드는 이유:

```text
FastAPI가 필요한 데이터만 받게 하기 위해
Date를 문자열로 바꾸기 위해
불필요한 author, createdAt 같은 값을 빼기 위해
```

# 15. React 로딩 state

AI 분석은 시간이 걸릴 수 있다.

그래서 로딩 상태가 필요하다.

```ts
const [isAnalyzing, setIsAnalyzing] = useState(false);
```

분석 시작:

```ts
setIsAnalyzing(true);
```

분석 종료:

```ts
setIsAnalyzing(false);
```

화면에서는 이렇게 쓴다.

```tsx
<button disabled={isAnalyzing}>
  {isAnalyzing ? '분석 중...' : 'AI 분석'}
</button>
```

뜻:

```text
분석 중이면 버튼을 비활성화하고 "분석 중..."이라고 보여준다.
분석 중이 아니면 "AI 분석"이라고 보여준다.
```

---

# 16. disabled

HTML 버튼에는 `disabled` 속성이 있다.

```tsx
<button disabled={isAnalyzing}>
```

뜻:

```text
isAnalyzing이 true이면 버튼을 누를 수 없게 한다.
```

왜 필요한가?

```text
AI 분석 버튼을 여러 번 연속 클릭하면 요청이 여러 번 나갈 수 있다.
분석 중에는 버튼을 잠가서 중복 요청을 막는다.
```

# 17. try / catch / finally

API 요청은 실패할 수 있다.

그래서 `try / catch / finally`를 쓴다.

```ts
try {
  const result = await analyzePost(postId);
  setAnalysis(result);
} catch (error) {
  setAnalysisError('AI 분석에 실패했습니다.');
} finally {
  setIsAnalyzing(false);
}
```

뜻:

```text
try
-> 일단 분석 요청을 시도한다.

catch
-> 실패하면 에러 메시지를 저장한다.

finally
-> 성공하든 실패하든 로딩 상태를 끝낸다.
```

`finally`가 중요한 이유:

```text
요청이 실패해도 버튼이 계속 "분석 중..." 상태로 남으면 안 된다.
```
