## 1. 새 우선순위

이제 우선순위는 이렇게 바꾼다.

```text
1순위: React에서 회원가입 -> 로그인 -> 글 작성 -> 목록/상세 연결
2순위: 제목/운동명 검색
3순위: AI 분석 1차
4순위: RAG 최소 구현
5순위: MCP 최소 구현
6순위: Agent workflow 최소 구현
7순위: 댓글 / 태그 / 페이징 같은 없어도 되는 기능 보강
8순위: README / 발표자료 / 데모 안정화
```

이번 과제의 핵심은 일반 게시판 기능보다 AI 응용 기능이다.

하지만 제목/운동명 검색은 AI 기능 전에 먼저 하는 것이 맞다.

이유는 간단하다.

```text
AI가 이전 운동 기록을 참고하려면
운동명으로 기록을 찾는 흐름이 먼저 있어야 한다.
```

따라서 금요일에는 React 연결을 끝낸 뒤, 제목/운동명 검색까지 닫는다.

댓글/태그/페이징은 AI MVP 이후, 월요일 잉여 처리일에 구현한다.

## 3. AI 기능 최소 기준

## 3.1 AI 분석 최소 기준

AI 분석은 아래 결과를 보여주면 된다.

```text
지난 기록 대비 발전 여부
무게 유지/증량 판단
다음 운동 목표
짧은 요약
```

예:

```text
지난번 벤치프레스는 60kg 8/7/6이었고,
이번에는 60kg 8/8/7입니다.

반복 수가 늘었으므로 발전하고 있습니다.
아직 8/8/8을 모두 채우지는 못했으므로,
다음 운동에서는 60kg으로 8/8/8을 목표로 하는 것이 좋습니다.
```

## 3.2 RAG 최소 기준

완성형 vector RAG보다 먼저 구조화 검색 기반 RAG로 간다.

```text
로그인한 사용자
+ 같은 운동명
+ 최근 기록 N개
```

이전 기록을 검색해서 AI prompt에 넣고 답변을 만든다.

pgvector는 시간이 되면 붙인다.

시간이 부족하면 README에 이렇게 남긴다.

```text
현재는 구조화 검색 기반 RAG로 구현했습니다.
추후 pgvector embedding 검색으로 확장할 수 있습니다.
```

## 3.3 MCP 최소 기준

MCP는 외부 도구 1개만 명확하게 만든다.

가장 현실적인 선택:

```text
운동명 정규화 tool
```

예:

```text
"벤치", "bench press", "벤치프레스"
        ↓
"벤치프레스"
```

이 tool은 AI 분석 전에 호출된다.

## 3.4 Agent 최소 기준

완전 자율 Agent를 만들지 않는다.

정해진 workflow로 구현한다.

```text
1. 현재 게시글 조회
2. 운동명 정규화 tool 호출
3. 이전 기록 검색
4. AI 분석 요청
5. 다음 운동 목표 추천
6. 결과 반환 또는 저장
```

Agent라고 설명할 포인트:

```text
한 번의 요청 안에서 여러 단계를 순서대로 실행하고,
각 단계의 결과를 상태처럼 들고 가며,
최종 추천 결과를 만든다.
```
