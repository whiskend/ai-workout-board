# 0. 오늘 새로 만나는 문법

오늘 핵심 문법과 개념은 아래다.

```text
fetch
headers
JSON.stringify
response.ok
response.json
localStorage
useState
useEffect
FormEvent
onChange
onSubmit
useNavigate
useParams
Link
map
conditional rendering
type import
generic <T>
Promise<T>
```

오늘의 핵심 흐름:

```text
input 값을 state에 저장한다.
        ↓
form 제출을 막는다.
        ↓
fetch로 백엔드 API를 호출한다.
        ↓
응답 JSON을 객체로 바꾼다.
        ↓
로그인 성공 시 token을 저장한다.
        ↓
token을 Authorization header에 넣어 보호 API를 호출한다.
        ↓
받아온 데이터를 state에 저장한다.
        ↓
state를 화면에 렌더링한다.
```

# 1. fetch

`fetch`는 브라우저에서 HTTP 요청을 보내는 함수다.

예:

```ts
const response = await fetch('http://localhost:3000/posts');
```

뜻:

```text
백엔드의 /posts API로 요청을 보낸다.
```

아무 옵션을 안 주면 기본은 GET 요청이다.

```ts
fetch('http://localhost:3000/posts')
```

이건:

```text
GET /posts
```

와 같다.

---

# 2. fetch 옵션

POST 요청처럼 body를 보내야 할 때는 옵션 객체를 넣는다.

```ts
await fetch('http://localhost:3000/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'test@example.com',
    password: '12345678',
  }),
});
```

각 부분 의미:

```text
method    요청 방식
headers   요청에 대한 추가 정보
body      백엔드로 보낼 실제 데이터
```

---

# 3. headers

headers는 요청에 붙이는 부가 정보다.

JSON을 보낼 때:

```ts
headers: {
  'Content-Type': 'application/json',
}
```

뜻:

```text
내가 보내는 body는 JSON 형식이다.
```

로그인 token을 보낼 때:

```ts
headers: {
  Authorization: `Bearer ${token}`,
}
```

뜻:

```text
나는 로그인한 사용자다.
증거로 accessToken을 보낸다.
```

---

# 4. JSON.stringify

JavaScript 객체는 그대로 HTTP body에 넣을 수 없다.

그래서 JSON 문자열로 바꾼다.

```ts
const body = JSON.stringify({
  email: 'test@example.com',
  password: '12345678',
});
```

결과:

```json
"{\"email\":\"test@example.com\",\"password\":\"12345678\"}"
```

쉽게 말하면:

```text
JavaScript 객체 -> JSON 문자열
```

백엔드는 이 JSON 문자열을 다시 객체처럼 해석해서 DTO에 넣는다.

---

# 5. response.ok

`fetch`는 400, 401, 403, 404 응답이 와도 JavaScript 에러를 자동으로 던지지 않는다.

그래서 직접 확인해야 한다.

```ts
if (!response.ok) {
  throw new Error('API 요청에 실패했습니다.');
}
```

`response.ok`는 HTTP 상태 코드가 200번대면 true다.

```text
200 OK       true
201 Created  true
400 Bad Request false
401 Unauthorized false
403 Forbidden false
404 Not Found false
500 Server Error false
```

---

# 6. response.json

백엔드가 JSON으로 응답하면 React에서 객체로 바꿔야 한다.

```ts
const data = await response.json();
```

뜻:

```text
응답 body의 JSON 문자열을 JavaScript 객체로 바꾼다.
```

예:

```json
{
  "id": 1,
  "title": "가슴 운동"
}
```

이 응답은 React에서 이렇게 쓸 수 있게 된다.

```ts
data.id
data.title
```

---

# 7. localStorage

`localStorage`는 브라우저에 문자열 데이터를 저장하는 공간이다.

로그인 성공 후 accessToken을 저장할 때 쓴다.

```ts
localStorage.setItem('accessToken', accessToken);
```

꺼낼 때:

```ts
const token = localStorage.getItem('accessToken');
```

삭제할 때:

```ts
localStorage.removeItem('accessToken');
```

주의:

```text
localStorage는 편하지만 보안상 완벽한 저장소는 아니다.
과제 MVP에서는 사용해도 충분하다.
```

---

# 8. useState

`useState`는 React 컴포넌트 안에서 변하는 값을 저장한다.

예:

```tsx
const [email, setEmail] = useState('');
```

뜻:

```text
email이라는 상태값을 만든다.
초기값은 빈 문자열이다.
email을 바꿀 때는 setEmail을 쓴다.
```

input과 연결하면:

```tsx
<input value={email} onChange={(event) => setEmail(event.target.value)} />
```

흐름:

```text
사용자가 input에 입력
        ↓
onChange 실행
        ↓
setEmail 실행
        ↓
email state 변경
        ↓
화면 다시 렌더링
```

---

# 9. controlled input

React에서 input 값을 state로 관리하는 방식을 controlled input이라고 한다.

```tsx
const [title, setTitle] = useState('');

<input value={title} onChange={(event) => setTitle(event.target.value)} />
```

뜻:

```text
input의 현재 값은 title state가 결정한다.
input이 바뀌면 setTitle로 state를 바꾼다.
```

즉 input이 혼자 값을 들고 있는 게 아니라 React state가 값을 관리한다.

---

# 10. FormEvent

폼 제출 함수의 event 타입이다.

```tsx
import { FormEvent } from 'react';

async function handleSubmit(event: FormEvent<HTMLFormElement>) {
  event.preventDefault();
}
```

뜻:

```text
이 event는 HTML form에서 발생한 submit event다.
```

처음에는 타입을 외우기보다 이렇게 기억하면 된다.

```text
form onSubmit 함수에는 FormEvent<HTMLFormElement>
```

---

# 11. event.preventDefault

HTML form은 기본적으로 submit하면 페이지를 새로고침한다.

React 앱에서는 보통 새로고침하면 안 된다.

그래서 막는다.

```ts
event.preventDefault();
```

뜻:

```text
브라우저의 기본 form 제출 동작을 막는다.
```

이걸 안 쓰면:

```text
버튼 클릭
        ↓
페이지 새로고침
        ↓
React state 초기화
        ↓
원하는 API 흐름이 깨짐
```

---

# 12. useEffect

`useEffect`는 화면이 렌더링된 뒤 어떤 작업을 실행할 때 쓴다.

게시글 목록 페이지에서는 처음 화면이 열릴 때 API를 호출해야 한다.

```tsx
useEffect(() => {
  async function loadPosts() {
    const data = await getPosts();
    setPosts(data);
  }

  loadPosts();
}, []);
```

마지막 `[]` 의미:

```text
이 컴포넌트가 처음 나타날 때 한 번만 실행한다.
```

---

# 13. useNavigate

`useNavigate`는 코드로 페이지를 이동할 때 쓴다.

예:

```tsx
const navigate = useNavigate();

navigate('/posts');
```

뜻:

```text
/posts 페이지로 이동한다.
```

로그인 성공 후:

```ts
storeToken(result.accessToken);
navigate('/posts');
```

흐름:

```text
로그인 성공
        ↓
token 저장
        ↓
게시글 목록으로 이동
```

---

# 14. useParams

`useParams`는 URL param을 꺼낼 때 쓴다.

라우트가 이렇게 되어 있다고 하자.

```tsx
<Route path="/posts/:id" element={<PostDetailPage />} />
```

사용자가 이 주소로 들어오면:

```text
/posts/7
```

코드에서 이렇게 꺼낸다.

```tsx
const params = useParams();
const postId = Number(params.id);
```

결과:

```text
params.id = "7"
postId = 7
```

주의:

```text
URL에서 꺼낸 값은 문자열이다.
숫자로 쓰려면 Number()로 바꿔야 한다.
```

---

# 15. Link

`Link`는 React Router에서 페이지 이동 링크를 만들 때 쓴다.

```tsx
<Link to="/posts">운동 기록</Link>
```

일반 HTML의 `<a>`와 비슷하지만, React 앱에서는 `Link`를 쓰는 게 좋다.

이유:

```text
페이지 전체 새로고침 없이 React 내부에서 화면만 바뀐다.
```

게시글 상세 링크:

```tsx
<Link to={`/posts/${post.id}`}>{post.title}</Link>
```

---

# 16. map

배열을 화면 목록으로 바꿀 때 쓴다.

```tsx
posts.map((post) => (
  <li key={post.id}>{post.title}</li>
))
```

뜻:

```text
posts 배열의 각 post를 li 태그로 바꾼다.
```

React에서 배열 렌더링을 할 때는 `key`가 필요하다.

```tsx
key={post.id}
```

`key`는 React가 각 항목을 구분하기 위해 사용한다.

---

# 17. conditional rendering

조건에 따라 화면을 다르게 보여주는 것이다.

예:

```tsx
{error && <p>{error}</p>}
```

뜻:

```text
error가 있으면 p 태그를 보여준다.
error가 없으면 아무것도 보여주지 않는다.
```

로딩 처리:

```tsx
if (isLoading) {
  return <main>게시글을 불러오는 중입니다.</main>;
}
```

뜻:

```text
로딩 중이면 원래 화면 대신 로딩 문구를 보여준다.
```

# 18. import type

TypeScript 타입만 가져올 때는 `import type`을 쓴다.

```ts
import type { Post } from '../types/post';
```

뜻:

```text
Post는 실행 중에 필요한 값이 아니라 TypeScript 검사에만 필요한 타입이다.
```

일반 import:

```ts
import { getPosts } from '../api/posts';
```

이건 실제 함수가 실행 중에 필요하다.

타입 import:

```ts
import type { Post } from '../types/post';
```

이건 컴파일 후 JavaScript에서는 사라진다.

---

# 19. generic `<T>`

공통 API 함수에서 이런 문법을 쓴다.

```ts
export async function apiRequest<T>(...): Promise<T> {
  ...
}
```

여기서 `<T>`는 “응답 타입을 나중에 정하겠다”는 뜻이다.

예:

```ts
apiRequest<User>('/auth/me')
```

이 경우:

```text
T = User
```

```ts
apiRequest<Post[]>('/posts')
```

이 경우:

```text
T = Post[]
```

즉 `apiRequest` 하나로 여러 API 응답 타입을 처리하기 위해 쓴다.

---

# 20. Promise<T>

비동기 함수는 바로 값을 돌려주는 게 아니라 Promise를 돌려준다.

```ts
async function getPosts(): Promise<Post[]> {
  ...
}
```

뜻:

```text
지금 당장 Post[]가 있는 것은 아니지만,
비동기 작업이 끝나면 Post[]가 나온다.
```

사용할 때는 `await`한다.

```ts
const posts = await getPosts();
```

---

# 21. Number()

HTML input 값은 항상 문자열이다.

예:

```tsx
const [weightKg, setWeightKg] = useState('');
```

사용자가 `60`을 입력해도 state에는 문자열 `"60"`으로 들어간다.

백엔드는 숫자를 기대한다.

그래서 보낼 때 바꾼다.

```ts
weightKg: Number(weightKg)
```

주의:

```text
Number('60') = 60
Number('') = 0
Number('abc') = NaN
```

나중에는 입력 검증을 더 추가하는 게 좋다.
