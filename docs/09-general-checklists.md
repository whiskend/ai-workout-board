# 09. General Checklists

이 문서는 어떤 프로젝트에서도 사용할 수 있는 일반 체크리스트다.

---

## 1. 작업 시작 체크리스트

```text
- 지금 작업이 기획/설계/구현/리뷰/테스트 중 어디인지 알고 있는가?
- Codex가 사용자의 현재 역할을 먼저 말했는가?
- 작업 범위가 한 기능으로 좁혀졌는가?
- 수정할 파일 목록을 받았는가?
- 변경하지 않을 영역을 정했는가?
- 사용자가 승인하기 전 코드 수정이 시작되지 않았는가?
```

---

## 2. 기획 체크리스트

```text
- 이 기능이 해결하는 문제가 명확한가?
- 핵심 사용자가 명확한가?
- MVP에 필요한지 판단했는가?
- 지금 하지 않을 것을 정했는가?
- 성공 기준이 있는가?
```

---

## 3. 설계 체크리스트

```text
- 가능한 설계안을 비교했는가?
- API 영향이 정리되었는가?
- DB 영향이 정리되었는가?
- 화면 영향이 정리되었는가?
- 외부 서비스 영향이 정리되었는가?
- 기존 구조를 유지할 수 있는가?
- 새로 복잡한 구조를 만들 필요가 있는가?
- 나중에 확장할 수 있는가?
```

---

## 4. 구현 체크리스트

```text
- 승인된 파일만 수정했는가?
- 관련 없는 리팩터링을 하지 않았는가?
- 새 패키지를 불필요하게 추가하지 않았는가?
- DB schema를 승인 없이 바꾸지 않았는가?
- 기존 API 응답 형식을 깨지 않았는가?
- 에러 처리가 있는가?
```

---

## 5. 리뷰 체크리스트

```text
- 변경사항을 파일별로 설명할 수 있는가?
- 설계 결정과 맞는가?
- 인증이 필요한 곳에 인증이 있는가?
- 권한 검사가 필요한 곳에 권한 검사가 있는가?
- 내 데이터와 남의 데이터가 섞이지 않는가?
- 기존 기능이 깨지지 않았는가?
- 문서와 코드 상태가 맞는가?
```

---

## 6. 테스트 체크리스트

```text
- 빌드/타입 체크를 실행했는가?
- 핵심 성공 케이스를 확인했는가?
- 실패 케이스를 확인했는가?
- 실제 사용자 흐름으로 확인했는가?
- 에러가 났을 때 화면/서버가 깨지지 않는가?
```

---

## 7. 커밋 체크리스트

```text
- git status를 확인했는가?
- 의도하지 않은 파일 변경이 없는가?
- 테스트 결과를 기록했는가?
- 문서가 최신 상태인가?
- 커밋 메시지가 변경 이유를 설명하는가?
```

커밋 메시지 예시:

```text
feat: add basic analysis request flow
fix: restrict related record query to past items
refactor: separate API client type
chore: update project docs for current state
```
