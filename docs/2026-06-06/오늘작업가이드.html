2. Git 연결 상태 확인

저장소가 제대로 연결됐는지 네 손으로 확인한다.

입력할 명령어

git status
git branch --show-current
git remote -v

정상 결과

현재 브랜치가 main으로 보인다.
origin https://github.com/whiskend/ai-workout-board.git 이 보인다.
작업 중인 변경 파일이 없거나, 오늘 작업 가이드 HTML만 변경 파일로 보인다.

해석

  • git status: 지금 Git이 보는 변경 사항을 보여준다.
  • git branch --show-current: 지금 내가 어느 작업 공간에 있는지 보여준다.
  • git remote -v: 내 로컬 프로젝트가 어느 GitHub 저장소와 연결됐는지 보여준다.

4. Docker로 DB 실행하기

백엔드가 데이터를 저장하려면 DB가 필요하다. 오늘은 PostgreSQL을 직접 설치하지 않고 Docker로 실행한다.

먼저 Docker Desktop 실행

macOS에서 Docker Desktop 앱을 켠다. 상단 메뉴바에 Docker 아이콘이 보이고 실행 중이어야 한다.

입력할 명령어

docker compose up -d

확인 명령어

docker compose ps

정상 결과

workout-board-postgres 컨테이너가 Up 또는 healthy 상태로 보인다.

실패했을 때

상황 원인 다음 행동
Cannot connect to the Docker daemon Docker Desktop이 꺼져 있음 Docker Desktop을 켜고 다시 실행
port is already allocated 이미 5432 포트를 쓰는 PostgreSQL이 있음 나에게 오류 메시지를 그대로 보여주기
이미지가 다운로드되지 않음 네트워크 문제 인터넷 확인 후 다시 실행

5. DB가 진짜 켜졌는지 확인하기

컨테이너가 떠 있는 것과 DB가 실제로 준비된 것은 다를 수 있다. 로그를 확인한다.

입력할 명령어

해석

이 문장은 PostgreSQL이 요청을 받을 준비가 됐다는 뜻이다. 월요일에 NestJS가 이 DB에 연결하게 된다.

7. 오늘 작업을 Git에 남기기

Docker 실행 확인까지 끝났다면, 오늘 만든 작업 가이드 문서를 Git에 남긴다. 이 단계는 네가 직접 커밋 연습을 하기 위한 것이다.

변경 파일 확인

오늘 작업 가이드만 추가

커밋 생성

git commit -m "docs: 오늘 작업 가이드 보강"

GitHub에 올리기

git push -u origin feat/1-init

정상 결과

GitHub에 feat/1-init 브랜치가 올라간다.
브라우저에서 Pull Request를 만들 수 있다는 안내가 보일 수 있다.
주의

커밋할 때 다른 파일이 같이 잡히면 바로 커밋하지 말고 git status --short 결과를 확인한다. 의도하지 않은 파일을 함께 올리지 않는 습관이 중요하다.
