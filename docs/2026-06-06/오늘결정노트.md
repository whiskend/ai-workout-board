## 1. 오늘 확정해도 되는 것

## 1.1 서비스 주제

확정:

> 운동 기록을 게시글로 남기면, AI가 이전 기록과 비교해 점진적 과부하 관점에서 다음 운동 목표를 추천해주는 운동 인증 게시판

이 주제는 유지한다.  
갈아엎지 않는다.

다만 표현은 “운동 기록 게시판”이 아니라 다음처럼 가져간다.

> AI 기반 점진적 과부하 코칭 게시판

---

## 1.2 핵심 사용자

확정:

- 헬스를 시작한 지 얼마 안 된 초보자
- 운동 기록은 남기지만 다음 목표를 정하기 어려운 사람
- 무게를 언제 올려야 할지 감으로 판단하는 사람
- 혼자 운동해서 기록을 봐줄 사람이 없는 사람

이 서비스는 전문 트레이너용 앱이 아니다.  
부상 진단이나 의학적 처방도 하지 않는다.

---

## 1.3 핵심 데모 흐름

확정:

1. 로그인
2. 지난 운동 기록 확인
3. 오늘 운동 기록 작성
4. 운동명, 무게, 세트별 반복 수 입력
5. 게시글 저장
6. AI 분석 실행
7. 이전 기록과 비교
8. 다음 운동 목표 추천
9. 댓글로 응원

발표에서는 이 흐름만 안정적으로 보여주면 된다.

---

## 1.4 기술 스택 큰 방향

확정:

| 영역 | 결정 |
| --- | --- |
| Frontend | React + TypeScript |
| Backend | NestJS + TypeScript |
| AI Server | FastAPI + Python |
| Database | PostgreSQL + pgvector |
| Auth | JWT |
| Infra | Docker Compose, AWS는 후반 적용 |

이유:

- React는 과제 요구사항이다.
- NestJS는 게시판, 인증, CRUD를 구조적으로 만들기 좋다.
- FastAPI는 AI 기능을 Python 생태계에서 구현하기 좋다.
- PostgreSQL은 게시판 데이터와 운동 기록 데이터에 적합하다.
- pgvector는 RAG용 이전 기록 검색에 쓴다.

## 2. 오늘 임시로 정할 것

완전히 이해하지 못해도 일단 이렇게 간다.  
나중에 바꿀 수 있지만, 지금은 이 결정으로 시작한다.

## 2.1 ORM

임시 결정:

- NestJS는 Prisma 사용
- FastAPI는 처음에는 raw SQL 또는 SQLAlchemy 중 편한 방식 사용

이유:

- Prisma는 TypeScript에서 DB schema를 읽기 좋게 관리할 수 있다.
- 처음 NestJS를 배우는 입장에서는 TypeORM보다 Prisma가 빠르게 이해될 가능성이 높다.
- FastAPI는 AI 서버이므로 DB 전체를 다루기보다 AI 관련 테이블만 최소 접근한다.

---

## 2.2 DB 소유권

임시 결정:

- 일반 서비스 데이터는 NestJS가 주인
- AI 분석 데이터는 FastAPI가 주인

NestJS 담당:

- users
- workout_posts
- workout_exercises
- workout_sets
- comments
- tags
- post_tags

FastAPI 담당:

- post_embeddings
- ai_analyses
- agent_runs
- agent_steps
- mcp_tool_logs

주의:

두 서버가 같은 테이블을 마음대로 수정하지 않는다.

---

## 2.3 AI 분석 방식

임시 결정:

처음에는 완전한 LLM Agent보다 “정해진 workflow”로 만든다.

```text
현재 게시글 조회
-> 이전 운동 기록 조회
-> 필요하면 유사 기록 검색
-> 필요하면 MCP tool 호출
-> 점진적 과부하 규칙 적용
-> LLM으로 사용자 친화적 문장 생성
-> 분석 결과 저장
```

이걸 Agent라고 설명한다.

완전 자율 Agent는 이번 과제에서 하지 않는다.

---

## 2.4 MCP tool

임시 결정:

MCP tool은 1개만 구현한다.

1순위:

- 운동명 정규화 또는 운동 정보 조회 tool

예:

```text
사용자 입력: 벤치
정규화 결과: 벤치프레스 / Bench Press
운동 부위: 가슴
```

외부 API가 어려우면 fallback 데이터를 둔다.
