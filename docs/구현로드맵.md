## 0. 로드맵 원칙

이번 과제의 핵심은 “운동 기록 게시판 + AI 분석”이 실제로 끝까지 동작하는 것입니다.

우선순위는 다음 순서입니다.

1. 운동 기록 게시글이 저장되고 조회된다.
2. 로그인한 사용자별 기록이 구분된다.
3. 이전 운동 기록과 현재 운동 기록을 비교할 수 있다.
4. AI가 다음 운동 목표를 추천한다.
5. RAG, MCP, Agent를 서비스 흐름 안에 최소 1개씩 연결한다.
6. 데모와 README로 설명 가능하게 만든다.

처음부터 예쁜 UI, 복잡한 그래프, 완전한 Agent, AWS 완성 배포를 목표로 잡지 않습니다. 핵심 데모가 먼저입니다.

## 5. 리스크와 대응

| 리스크 | 대응 |
| --- | --- |
| NestJS와 FastAPI를 동시에 하느라 시간이 부족함 | 먼저 NestJS 게시판을 닫고, FastAPI는 AI 기능만 맡긴다. |
| pgvector 설정이 오래 걸림 | 구조화 DB 검색으로 먼저 이전 기록 비교를 만들고, pgvector는 그다음 붙인다. |
| MCP가 막힘 | 외부 API 호출 tool 하나만 최소 구현하고 로그를 남긴다. |
| Agent가 복잡해짐 | 완전 자율 Agent 대신 정해진 workflow로 구현한다. |
| OpenAI API 실패 | 데모용 fallback 응답과 스크린샷을 준비한다. |
| AWS 배포가 늦어짐 | Docker Compose 로컬 실행을 우선 완성하고, AWS는 시도/계획까지 문서화한다. |
| UI 시간이 부족함 | 예쁜 화면보다 운동 기록 표와 AI 결과 표시를 우선한다. |

## 7. 최종 목표

최종적으로 보여줄 흐름은 하나입니다.

```text
로그인
-> 지난 운동 기록 확인
-> 오늘 운동 기록 작성
-> AI 분석 실행
-> 이전 기록과 비교
-> 다음 운동 목표 추천
-> 댓글로 응원
```

이 흐름이 안정적으로 동작하면 과제의 핵심은 성공입니다.
