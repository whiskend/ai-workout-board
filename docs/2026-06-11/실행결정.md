## 2. 기술 결정

## 2.1 비밀번호 저장

비밀번호는 DB에 그대로 저장하지 않는다.

사용 도구:

```text
bcrypt
```

저장 방식:

```text
password -> bcrypt.hash(password, 10) -> passwordHash
```

DB에는 `passwordHash`만 저장한다.

## 2.2 로그인 방식

로그인 성공 시 JWT를 발급한다.

응답 예시:

```json
{
  "accessToken": "jwt-token",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "nickname": "운동초보"
  }
}
```

## 2.3 JWT 저장 위치

오늘 백엔드 기준으로는 토큰을 발급하는 것까지만 중요하다.

React 연결 시에는 우선 `localStorage`에 저장한다.

```text
localStorage는 MVP용으로 사용한다.
보안적으로 더 좋은 방식은 나중에 README 한계점에 적는다.
```

## 2.4 게시글 작성자 연결

현재:

```text
demo user로 게시글 작성
```

변경:

```text
JWT 토큰의 userId로 게시글 작성
```

## 2.5 권한 처리

수정/삭제는 작성자만 가능하게 한다.

```text
post.authorId === request.user.userId
```

이 조건이 맞으면 허용한다.

다르면:

```text
403 Forbidden
```
