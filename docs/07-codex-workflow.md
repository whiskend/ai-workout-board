# 07. Codex Workflow

이 문서는 Codex와 함께 작업할 때 쓰는 프롬프트 모음이다.

모든 프롬프트의 핵심은 같다.

```text
아직 코드 수정하지 마.
먼저 내가 지금 어떤 역할을 해야 하는지 알려줘.
```

---

## 1. 첫 시작 프롬프트

```text
AGENTS.md와 docs/01-project-context.md를 먼저 읽어줘.

아직 코드를 수정하지 마.
먼저 이 프로젝트에서 내가 지금 어떤 역할을 해야 하는지 알려줘.

아래 형식으로만 답해줘.

역할 확인:
- 현재 단계:
- 사용자의 역할:
- Codex의 역할:
- 지금 사용자가 해야 할 일:
- 아직 하면 안 되는 일:

프로젝트 이해 요약:
- 프로젝트 목적:
- 현재 구조:
- 주요 기능:
- 위험하게 건드리면 안 되는 영역:
- 다음 작업 후보 3개:
```

---

## 2. 기획 요청 프롬프트

```text
아직 코드 수정하지 마.
지금은 기획 단계야.

이번 기능은 [기능 이름]이야.
내가 기획자로서 결정할 수 있게 아래를 정리해줘.

역할 확인:
- 현재 단계:
- 사용자의 역할:
- Codex의 역할:
- 지금 사용자가 해야 할 일:
- 아직 하면 안 되는 일:

기획 정리:
1. 이 기능이 해결하는 문제
2. 핵심 사용자
3. MVP에 꼭 필요한지
4. 필수 기능 / 나중 기능 / 제외 기능
5. 성공 기준
6. 내가 결정해야 할 것
```

---

## 3. 설계 요청 프롬프트

```text
아직 코드 수정하지 마.
지금은 설계 단계야.

이번 기능은 [기능 이름]이야.
내가 설계자로서 결정할 수 있게 아래를 정리해줘.

역할 확인:
- 현재 단계:
- 사용자의 역할:
- Codex의 역할:
- 지금 사용자가 해야 할 일:
- 아직 하면 안 되는 일:

설계 정리:
1. 가능한 설계안 2~3개
2. 각 설계안의 장단점
3. 추천안
4. 추천 이유
5. 수정될 파일
6. 새로 만들 파일
7. 변경하지 말아야 할 영역
8. API 영향
9. DB 영향
10. 테스트 방법
11. 내가 결정해야 할 것
```

---

## 4. 구현 승인 프롬프트

```text
좋아. [A안/B안/선택한 방향]으로 진행해.

단, 이번 작업 범위는 아래로 제한해.

작업 범위:
- 
- 

금지:
- DB schema 변경하지 마.
- 새 패키지 추가하지 마.
- 기존 API 응답 형식 바꾸지 마.
- 관련 없는 파일 수정하지 마.

작업 후 아래 형식으로 보고해줘.

완료한 작업:
수정한 파일:
새로 만든 파일:
변경하지 않은 영역:
테스트한 명령어:
내가 직접 확인해야 할 것:
다음 단계 추천:
```

필요한 금지 조건은 프로젝트 상황에 맞게 수정한다.

---

## 5. 리뷰 요청 프롬프트

```text
아직 코드 수정하지 마.
지금은 리뷰 단계야.

현재 변경사항을 기준으로 아래를 검토해줘.

역할 확인:
- 현재 단계:
- 사용자의 역할:
- Codex의 역할:
- 지금 사용자가 해야 할 일:
- 아직 하면 안 되는 일:

리뷰 결과:
1. 변경된 파일별 역할 요약
2. 설계상 맞는 부분
3. 위험한 부분
4. 지금 당장 고쳐야 하는 부분
5. 나중에 개선해도 되는 부분
6. 내가 직접 확인해야 할 부분
7. 테스트해야 할 시나리오
8. 커밋해도 되는 상태인지 판단

주의:
- 문제를 발견해도 바로 수정하지 마.
- 먼저 리뷰 결과만 보고해.
```

---

## 6. 테스트 요청 프롬프트

```text
지금은 테스트 단계야.
아직 코드 수정하지 마.

이번 변경사항을 내가 직접 확인할 수 있게 테스트 시나리오를 만들어줘.

역할 확인:
- 현재 단계:
- 사용자의 역할:
- Codex의 역할:
- 지금 사용자가 해야 할 일:
- 아직 하면 안 되는 일:

테스트 계획:
1. 실행해야 할 명령어
2. 성공 케이스
3. 실패 케이스
4. 엣지 케이스
5. 예상 결과
6. 실패했을 때 확인할 파일
```

---

## 7. 길을 잃었을 때 프롬프트

```text
지금 내가 뭘 하고 있는지 헷갈려.
아직 코드 수정하지 마.

현재 상황을 기준으로 아주 쉽게 정리해줘.

1. 지금 프로젝트는 어느 단계인지
2. 내가 지금 맡아야 할 역할은 무엇인지
3. 이 역할에서 내가 해야 할 일 3개
4. 아직 하면 안 되는 일 3개
5. Codex에게 다음으로 시킬 가장 작은 작업 1개
```

---

## 8. 문서 정리 요청 프롬프트

```text
아직 코드 수정하지 마.
지금은 문서 정리 단계야.

현재 구현 상태를 기준으로 README/docs에 반영해야 할 내용을 정리해줘.

1. 구현 완료로 적어도 되는 것
2. 아직 예정으로 적어야 하는 것
3. 과장하면 안 되는 표현
4. 발표에서 안전하게 말할 수 있는 문장
5. Mermaid나 표로 정리하면 좋은 흐름
```
